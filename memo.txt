#### Cloud BuildとCloud Runの連携がうまくいかない時の、直接アップロード方法
https://zenn.dev/ryu02/articles/e82b8816614ff3
# 初回だけ
gcloud artifacts repositories create nazo-doukou-repo --location=asia-northeast1 --repository-format=docker --project=nazo-doukou
gcloud auth configure-docker asia-northeast1-docker.pkg.dev

docker build -t asia-northeast1-docker.pkg.dev/nazo-doukou/nazo-doukou-repo/nazo-doukou-image:latest .
docker push asia-northeast1-docker.pkg.dev/nazo-doukou/nazo-doukou-repo/nazo-doukou-image:latest
# 初回だけ
gcloud run deploy --image asia-northeast1-docker.pkg.dev/nazo-doukou/nazo-doukou-repo/nazo-doukou-image:latest --platform=managed --project=nazo-doukou

#### Cloud StorageでCORS設定の対応
PROJECT_ID=nazo-doukou
# プロジェクトを確認
  gcloud config get-value project
# プロジェクトを変更
  gcloud config set project ${PROJECT_ID}
gsutil cors set cors_setting_prod.json gs://nazo-doukou-production-bucket
gsutil cors get gs://nazo-doukou-production-bucket

##### Cloud Domainsでドメインを買う場合 ####
＊お名前.comのやり方がネットに書かれてあるので、そもそもお名前.com使った方が楽だし安い

Cloud DNSにアクセスして、レコードセット > 標準を追加 で、Aレコード、AAAAレコードを追加する
さらにTXTレコードを追加する
サーチコンソールの、設定 > 所有権の確認にTXTの値がある
追加した後に、確認ボタンを押す
